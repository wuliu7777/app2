# 多平台实用工具集合 - 视频提取工具 设计方案

## 1. 概述
构建一个多平台（Windows, macOS, Android, iOS）的实用工具集合应用。首个功能为“全平台无水印视频提取”。采用 C/S 架构，自建开源后端服务解决视频解析难题，前端使用 Flutter 提供一致的跨平台体验。

## 2. 系统架构
*   **前端客户端**：Flutter (Dart)。负责跨平台 UI 展示、用户交互、调用系统播放器和文件下载系统。
*   **后端解析微服务**：Python/Node.js (基于开源项目如 Douyin_TikTok_Download_API 或 yt-dlp API)。提供统一的 RESTful API，接收原始链接，返回真实的无水印视频流地址、标题和封面。

## 3. UI 与交互流程设计
### 3.1 首页 (工具箱导航页)
*   **布局**：网格 (Grid) 布局，为未来添加新工具预留空间。
*   **内容**：目前展示一个“视频提取”的卡片/图标，点击进入视频提取页面。

### 3.2 视频提取页面
*   **输入区**：大尺寸的文本输入框，支持粘贴包含分享口令的混合文本。
*   **操作区**：醒目的“开始解析”按钮，附带 Loading 状态指示。
*   **结果区 (解析成功后展示)**：
    *   视频标题及封面图预览。
    *   “直接播放”按钮：调用内置播放器在线预览。
    *   “下载保存”按钮：将视频保存至本地相册或下载目录。

## 4. 核心数据流
1.  客户端正则提取用户输入中的 URL。
2.  客户端发起 HTTP GET/POST 请求至自建解析后端。
3.  后端处理反爬/签名/重定向，返回标准 JSON (含真实 video_url, cover_url, title)。
4.  客户端利用 video_url 进行流媒体播放或使用下载库进行二进制文件拉取。

## 5. 依赖库建议 (Flutter)
*   HTTP 请求：dio 或 http
*   视频播放：`video_player` (多端兼容)
*   文件下载与权限：dio (用于下载), path_provider, permission_handler, gal (保存至相册, 视具体需求选配)

2. 创建 Flutter 跨平台项目并完成 UI 框架搭建。
3. 联调后端 API，完成解析和下载流程。
